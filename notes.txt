LangChain is a framework for building applications powered by large language models (LLMs). It provides tools for chaining together LLM calls, managing memory, connecting to external data sources, and building agents that can use tools.

Retrieval Augmented Generation (RAG) is a technique that combines information retrieval with text generation. Instead of relying only on what the model learned during training, RAG systems first search a knowledge base for relevant information, then pass that information to the model as context before generating an answer.

The RAG pipeline typically has three main components: an embedding model that converts text into numerical vectors, a vector store (also called a vector database) that stores and searches these vectors efficiently, and a retriever that finds the most relevant chunks of text for a given query.

Common vector stores used in LangChain include FAISS, Chroma, and Pinecone. FAISS, developed by Meta, is popular for local development because it runs entirely on your own machine without needing a cloud account.

Embeddings capture the semantic meaning of text. Two sentences with similar meaning will have embedding vectors that are close together in vector space, even if they use completely different words. This is what allows RAG systems to find relevant information even when the user's question doesn't use the exact same wording as the source document.
